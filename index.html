DSA with JS - Day 2

1. Conditional Statements

Conditional statements control the flow of execution based on conditions.

2. Valid Voter

A person is considered a valid voter if they are 18 or older.

let age = 20;
if (age >= 18) {
    console.log("You are eligible to vote.");
} else {
    console.log("You are not eligible to vote.");
}

3. Shop Discount

Applying a discount based on the total bill amount.

let totalBill = 5000;
let discount = 0;

if (totalBill >= 5000) {
    discount = 0.2; // 20% discount
} else if (totalBill >= 3000) {
    discount = 0.15; // 15% discount
} else {
    discount = 0.1; // 10% discount
}

let finalAmount = totalBill - (totalBill * discount);
console.log("Final Amount after Discount: ", finalAmount);

4. Electric Bill

Electric bill based on unit consumption.

let units = 250;
let bill = 0;

if (units <= 100) {
    bill = units * 5;
} else if (units <= 200) {
    bill = (100 * 5) + ((units - 100) * 7);
} else {
    bill = (100 * 5) + (100 * 7) + ((units - 200) * 10);
}

console.log("Total Electric Bill: ", bill);

5. PKR Denomination

Breaking an amount into currency notes.

let amount = 1570;
let notes500 = Math.floor(amount / 500);
amount %= 500;
let notes100 = Math.floor(amount / 100);
amount %= 100;
let notes50 = Math.floor(amount / 50);
amount %= 50;
let notes10 = Math.floor(amount / 10);
amount %= 10;

console.log("500 PKR Notes: ", notes500);
console.log("100 PKR Notes: ", notes100);
console.log("50 PKR Notes: ", notes50);
console.log("10 PKR Notes: ", notes10);
console.log("Remaining Amount: ", amount);

6. Ternary Operator (?:)

Used to simplify conditional expressions.

let age = 20;
let eligibility = (age >= 18) ? "Eligible to vote" : "Not eligible to vote";
console.log(eligibility);

7. Nested Ternary Operator

Used for multiple conditions in a compact form.

let marks = 85;
let grade = (marks >= 90) ? "A+" : (marks >= 80) ? "A" : (marks >= 70) ? "B" : "C";
console.log("Grade: ", grade);

8. Switch Case

Used when multiple conditions depend on the same variable.

let day = 3;

switch (day) {
    case 1:
    case 2:
    case 3:
        console.log("It's a weekday.");
        break;
    case 6:
    case 7:
        console.log("It's the weekend!");
        break;
    default:
        console.log("Invalid day.");
}

9. Switch and Precision Issues

Switch case can't handle floating-point numbers due to precision issues.

let value = 2.5;

switch (value) {
    case 2.5:
        console.log("Matched");
        break;
    default:
        console.log("Not Matched");
}

In the above case, due to precision problems, switch-case may not work as expected for floating-point values.
